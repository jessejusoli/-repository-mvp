# Backlog

Use this file to capture work that has not started yet. Keep items tied to the current vertical goal unless they are explicitly marked as future work.

## Backlog Rules

- Every item must have a clear outcome.
- Every item must identify whether it supports the current version or a future version.
- Move active work to [TASK_MANAGER.md](TASK_MANAGER.md).
- Update [CHANGELOG.md](CHANGELOG.md) when completed work changes the project.
