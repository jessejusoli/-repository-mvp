# Requirements

This file defines what `[PROJECT_NAME]` must satisfy. It should describe needs and acceptance criteria, not implementation plans.

## Requirement Boundaries

- Requirements belong here.
- Candidate work derived from requirements belongs in [BACKLOG.md](../BACKLOG.md).
- Active execution belongs in [TASK_MANAGER.md](../TASK_MANAGER.md).
- Technical decisions belong in [architecture.md](architecture.md).
- Current version focus belongs in [../VERTICAL.md](../VERTICAL.md).

## Acceptance Criteria Rules

- Make acceptance criteria observable.
- Avoid implementation details unless the requirement depends on them.
- Link backlog items to requirement IDs when work starts.
- Update requirements when product intent changes.
